# Data Model: Live Agent Orchestration

## Session Reuse

The existing workspace `Session` remains the top-level container. This feature adds orchestration runs beneath that session instead of introducing a second top-level workspace concept.

### Relationships

- One `Session` has many `OrchestrationRun` records.
- One `Session` may have at most one active `OrchestrationRun` at a time for this feature scope.
- Reopening a session restores the latest selected orchestration run and its replayable agent state without contacting the model provider.

## Access Configuration

Represents the existing provider-access configuration already required for backend-driven live runs.

| Field | Type | Source | Rules |
|-------|------|--------|-------|
| `configured` | boolean | Derived from config | Must be true before an orchestration run can start |
| `provider_label` | text | Config | Optional display-safe identifier; never includes secrets |

### Validation

- If access is unavailable, orchestration submission is blocked with a plain-language error.
- Secrets remain server-side only and never appear in replayed events or canvas state.

## Orchestration Run

Represents one submitted goal and the full multi-agent execution it triggers.

| Field | Type | Source | Rules |
|-------|------|--------|-------|
| `id` | text | SQLite | Primary key; opaque identifier |
| `session_id` | text | SQLite | Required; foreign key to `Session` |
| `goal_text` | text | SQLite | Required; original developer goal |
| `mode` | text enum | SQLite | Required; `orchestration` for this feature |
| `state` | text enum | SQLite | Required; one of `accepted`, `active`, `completed`, `errored`, `halted`, `cancelled` |
| `started_at` | datetime | SQLite | Required |
| `completed_at` | datetime nullable | SQLite | Set when the run reaches a terminal state |
| `halt_reason_code` | text nullable | SQLite | Optional run-level failure classification |
| `halt_reason_message` | text nullable | SQLite | Optional display-safe explanation |
| `active_agent_count` | integer | Derived/runtime | Number of currently active agents |

### Validation

- Only one live orchestration run may exist at a time across the workspace.
- `goal_text` must be non-empty after trimming.
- Terminal failure messages must be display-safe and must not contain secrets.

### State Transitions

- Submit goal: `accepted`
- First agent spawn: `active`
- All required agents finish successfully: `completed`
- Run-level unrecoverable failure: `halted`
- Explicit run cancellation: `cancelled`
- Storage or provider failure before DAG execution stabilizes: `errored`

## Agent Execution

Represents one run-scoped agent instance visible as one canvas node.

| Field | Type | Source | Rules |
|-------|------|--------|-------|
| `id` | text | SQLite | Primary key; becomes frontend `agent_id` |
| `run_id` | text | SQLite | Required; foreign key to `OrchestrationRun` |
| `role` | text enum | SQLite | Required; one of `planner`, `coder`, `reviewer`, `tester`, `explainer` |
| `model` | text | SQLite | Required; resolved model used for that agent |
| `state` | text enum | SQLite | Required; one of `queued`, `assigned`, `thinking`, `streaming`, `completed`, `errored`, `cancelled`, `blocked` |
| `task_text` | text | SQLite | Required; agent-specific assignment text |
| `spawn_order` | integer | SQLite | Required; creation order inside the run |
| `started_at` | datetime nullable | SQLite | Set when the agent begins work |
| `completed_at` | datetime nullable | SQLite | Set on terminal state |
| `error_code` | text nullable | SQLite | Optional agent-scoped failure classification |
| `error_message` | text nullable | SQLite | Optional display-safe failure description |

### Validation

- `agent_id` must be stable across live delivery, reconnect, and replay.
- Each role appears at most once per run in this fixed DAG.
- `task_text` is prompt-only and must not embed file-system or shell instructions generated by the server.

### State Transitions

- Node created: `queued`
- Assignment delivered: `assigned`
- Agent begins processing: `thinking`
- Agent emits visible output: `streaming`
- Agent finishes successfully: `completed`
- Agent-scoped failure: `errored`
- Run-level stop while active: `cancelled`
- Upstream precondition prevents start: `blocked`

## Handoff Edge

Represents the fixed dependency relationship between two agent executions in the orchestration DAG.

| Field | Type | Source | Rules |
|-------|------|--------|-------|
| `run_id` | text | Derived | Scope key |
| `from_agent_id` | text | Derived | Upstream agent |
| `to_agent_id` | text | Derived | Downstream agent |
| `state` | text enum | Derived/runtime | One of `pending`, `started`, `completed`, `blocked` |

### Rules

- Planner hands off to both Coder and Tester.
- Coder and Tester both hand off to Reviewer.
- Reviewer hands off to Explainer.
- Handoff state changes are user-visible through explicit orchestration events rather than inferred only from agent terminal states.

## Orchestration Event

Append-only event emitted during a run for live delivery and replay.

| Field | Type | Source | Rules |
|-------|------|--------|-------|
| `run_id` | text | SQLite | Required |
| `sequence` | integer | SQLite | Required; strictly increasing per run |
| `agent_id` | text nullable | SQLite | Present for any agent-scoped event |
| `event_type` | text enum | SQLite | Required; one of `agent_spawned`, `agent_state_changed`, `task_assigned`, `handoff_start`, `handoff_complete`, `token`, `agent_error`, `run_complete`, `run_error` |
| `payload_json` | text | SQLite | Required; display-safe event body |
| `occurred_at` | datetime | SQLite | Required |
| `replayable` | boolean | SQLite | Required; true for all persisted orchestration events |

### Validation

- Event order is determined by `sequence`, not by timestamp alone.
- Every orchestration event in this mode carries `agent_id` when it relates to a node.
- `run_complete` and `run_error` are terminal run events and occur at most once per run.
- `agent_error` does not imply `run_error`; the run may continue if the DAG allows it.
- Event payloads must not contain secrets or hidden prompt content intended to stay internal.

## Agent Transcript Slice

Represents one visible chunk of output for a selected agent.

| Field | Type | Rules |
|-------|------|-------|
| `agent_id` | text | Required |
| `sequence` | integer | Required; tied to the parent run event order |
| `text` | text | Visible output only |
| `stream_index` | integer | Optional per-agent chunk order for frontend assembly |

### Rules

- Transcript slices remain ordered within each agent transcript.
- A selected node view is reconstructed by filtering replayed events on `agent_id` and transcript-bearing event types.

## Canvas Node Projection

Frontend-safe view model used by the live canvas.

| Field | Type | Rules |
|-------|------|-------|
| `agent_id` | text | Stable React Flow node key |
| `role` | text enum | Visible role badge |
| `label` | text | Visible node title |
| `state` | text enum | Current visible node state |
| `summary` | text | Latest display-safe summary or excerpt |
| `selected` | boolean | Derived from client state |
| `position` | object | Changes only on spawn-triggered layout recalculation |

### Invariants

- State-only updates do not change node coordinates.
- New node addition is the only event that triggers dagre layout.
- Reconnect and replay do not create duplicate node projections for the same `agent_id`.

## Selected Node View

Frontend-safe side-panel projection for one selected node.

| Field | Type | Rules |
|-------|------|-------|
| `agent_id` | text | Required |
| `role` | text enum | Visible role label |
| `state` | text enum | Current or final state |
| `task_text` | text | Agent assignment text |
| `transcript` | array | Ordered visible transcript slices |
| `error_message` | text nullable | Present only for display-safe failures |

### Rules

- Switching node selection changes the side panel without affecting run execution.
- Completed and errored nodes remain inspectable after the run ends.
